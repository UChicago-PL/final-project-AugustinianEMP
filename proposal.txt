**Project Proposal: BioFlow - A Lab Automation and Analysis Suite**

**Concept and Objectives**

The goal of this project is to automate the experimental loop within a cancer biology laboratory setting. 
Currently, much of the data management (including aliquot calculations, 96-well plate layout design, and 
post-experimental data analysis) is performed manually in Excel. This manual process is prone to human error 
and lacks the structural integrity required for complex experiments involving multiple primers and antibodies.

**BioFlow** will be a Haskell-based utility designed to model the physical constraints of the laboratory. 
By using ADTs to represent biological entities (like mRNA samples, proteins, and primers) and spatial structures 
(like 96-well plates), the system will programmatically prevent errors such as well-overflow or reagent depletion 
before the wet bench work begins.

---

**Project Milestones**

**Easy: The Aliquot Kernel**

* Implement a command-line tool to perform mass-aliquot calculations(C1V1 = C2V2).
* Support batch processing where a user can input a list of target concentrations 
and receive a calculated list of required stock, water, qScript volumes, and much more.

**Medium: Spatial Plate Layout Engine**

* Define a recursive ADT to represent a plate as a collection of wells, each containing a list of reagents.
* Implement logic to validate that an experiment fits within the physical dimensions of a standard plate 
(ex: 8x12 for a 96-well plate).
* Generate a visual text-based map of the plate layout to guide manual pipetting.

**Challenge: The Post-Processing Pipeline**

* Utilize parser combinators to read and validate machine-generated Excel/CSV result files.
* Implement a monadic error context (using Either) to flag invalid experimental results, 
such as CT values that fall outside a valid range or undefined RT values.
* Automate the generation of bar plots (Treatment vs. RQ value) using a Haskell plotting library.

---

**Additional Topics and Resources**

To achieve the challenge goals, I will need to explore some advanced Haskell topics beyond the current curriculum:

* Parser Combinators (Cassava/Megaparsec): I will need this to handle the messy formatting of 
machine-exported Excel/CSV files. I plan to learn this via the real world haskell documentation and online tutorials.
* Graphic Libraries (Chart): Generating the RQ bar plots will require learning the Chart or Gloss libraries.
* Lenses: To efficiently update specific wells within a complex nested plate structure, I may explore the 
Lens library to avoid deep record-syntax nesting.

I assume that I'll need some help when it comes to learning and using parser combinators 
to ensure the error handling is robust enough for real lab data.

---

**Inspiration**

This project is inspired by the daily workflow of a cancer biology lab, aiming to replace brittle 
Excel spreadsheets with the type-safety and mathematical rigor of functional programming.
